FEATURES IMPLEMENTED
================================================================================

MODE 1: OAuth2 Authorization
✅ Exchange authorization code for access token
✅ Fetch user profile from BikeIndex
✅ Encrypt token with AES-256-GCM
✅ Store encrypted token in users.bikeIndexToken
✅ Return user profile (username, email, bikeIndexId)

MODE 2: Cross-Register Bike
✅ Decrypt user's BikeIndex token
✅ Register bike with BikeIndex API
✅ Extract and store bikeIndexId in bikes collection
✅ Return BikeIndex bike URL

MODE 3: Verify Bike Serial
✅ Search BikeIndex by serial number (public API)
✅ Return array of matching bikes
✅ Include all bike details (brand, model, year, color, etc.)

GENERATION COMMANDS:
- Encryption Key: openssl rand -hex 32
- BikeIndex Credentials: https://bikeindex.org/account/applications

================================================================================
ENCRYPTION SPECIFICATION
================================================================================

ALGORITHM: AES-256-GCM
Key Size: 256 bits (32 bytes)
IV Size: 128 bits (16 bytes) - random per token
Auth Tag: 128 bits (16 bytes) - detects tampering
Mode: GCM (Galois/Counter Mode) - authenticated encryption

FORMAT: <iv_hex>:<auth_tag_hex>:<cipher_hex>

FEATURES:
✓ Authenticated encryption (confidentiality + authenticity)
✓ Random IV prevents pattern analysis
✓ GCM mode NIST SP 800-38D compliant
✓ 8 unit tests verify correctness
